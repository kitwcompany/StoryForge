# 高密度状态世界构建法

## 核心哲学

世界感不来自素材量，而来自**结构密度与元素交互**。

**克制即创造力**：硬约束逼出纪律，让少量元素自发产生复杂性、连续性和自主感。

小说不是作者全知讲述，而是读者在"状态网"中发现一个独立运转的世界。

## 核心组件

### 状态系统（State Vector）—— 小说的"物理引擎"

每个重要人物（尤其是主角群）维持一个动态状态向量：

- **身份/位置**
- **资源**（物质、金钱、知识、信息、物品）
- **关系旗标**（对关键人物的态度、恩怨、秘密）
- **历史/事件旗标**
- **心理/目标状态**

> **规则**：情节不是预设的，而是"当前状态 × 世界规则"的函数输出。任何行动或事件都应同时修改相关状态。

### 桥节点（Bridge Nodes）—— 世界压缩器

设计 **5–12 个**高连接度元素（人物、地点、关键物件、秘密/事件）。

这些节点必须承担多条线、多重功能，是不同命运的交汇点。

**效果**：读者在一条线看到另一条线的影子，世界瞬间变厚、有机且大于任何单视角。

### 事件回流机制（Convergence Loops）

允许分支，但必须在共享桥节点、共同危机、资源门槛或周期性事件上回流。

避免树状爆炸，保持世界统一性与命运交织感。

### 元素多功能重用（Multi-Role Density）

每一个重要元素（人物、地点、物件、习俗）都应至少承担 **3 种功能**：

1. **叙事功能**
2. **世界构建功能**
3. **象征/主题功能**
4. **玩法/门槛功能**（驱动人物行动或限制可能）

## 应用流程

### 阶段1：最小世界种子（Minimal Viable World）

- 先设计一个高密度"小切片"（例如一个港口、一场聚会、一个关键物件）
- 定义核心状态向量和 3–5 个桥节点
- **验证**：在这个小系统中，不同人物在相同状态下是否会自然产生不同行动与冲突？

### 阶段2：状态网扩张

- 扩展主角群，每人赋予独特初始状态，但共享部分桥节点
- 列出"状态触发表"：哪些状态组合会触发何种事件/对话/转折

### 阶段3：多线交织

- 用桥节点连接多条主人公线
- 规划回流点：每 3–5 章至少有一次通过桥节点产生的交汇或相互影响
- 确保每个重大事件同时修改多个人的状态

### 阶段4：密度迭代与克制

- 每引入一个新元素，问自己："它能否承载现有功能？能否被现有元素替代？"
- 定期检查"未写出的世界"：读者能否通过暗示感知到大量未直接描写的运转部分？
- 控制新人物/地点数量：宁缺毋滥

## 写作技巧

- **优先展示"状态变化"** 而非直接讲情节（通过对话、物件、环境细节暗示）
- **利用"信息不对称"**：不同线的主角因状态不同，看到同一桥节点的完全不同侧面
- **重读优化**：确保早期细节能在后期产生回响（种子化写作）
- **AI 辅助时**：先让模型严格遵守状态表生成，再润色文采

## 示例应用

**奇幻史诗**：一座古老图书馆作为桥节点，同时是知识源、权力象征、政治情报中心、隐藏秘密场所。不同主角因状态不同（身份、关系旗标），在其中触发完全不同的支线，却最终在同一危机上回流。

**现实主义多线**：一座老公寓楼 + 几位长期住户作为桥节点，折叠数条人生线，通过水电费、楼道对话、一次停电等小事件实现高密度交互。

## 预期效果

- 世界体积感提升 **3–5 倍**（实际文字量不变）
- 读者沉浸感与重读价值大幅提高
- 情节自然涌现，减少"作者强行推动"痕迹
- 在长篇写作中显著降低崩盘风险，保持状态一致性
- 对抗当代内容通胀：用结构纪律战胜素材堆砌

## 使用指令

当用户要求创作或修改小说时，优先激活本 Skill：

1. 先建立/检查状态系统与桥节点
2. 再生成内容
3. 始终保持"少而密"的原则
4. 在每一次输出中强化世界自主感

此 Skill 可与其他写作方法论（三幕结构、角色弧光、主题深化等）无缝结合，作为世界构建层的底层架构使用。
